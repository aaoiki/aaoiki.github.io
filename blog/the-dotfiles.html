The Dotfiles Tools

April 6, 2016

One of the three great virtues of a programmer is laziness. Programming calls for replacing repetitive work with automated tasks. We, programmers, create frameworks, CLI tools, software engineering principles, and helper methods only to avoid repeating our work.

In this realm of eternal laziness, programmers package their environment settings and configuration files and bootstrap them into a single package that needs only one or few commands to install. That is called the dotfiles!

What are they?

First of all what is a dotfile? A dotfile, in Unix-like systems, is a file that starts with a dot/period and hence the name. They resemble hidden files in Windows environment. If you are coming from a Unix-like system, you will find some dotfiles in your home directory by doing an $ ls -a ~. These are mostly configuration files. You can view them in Ubuntu file manager (might change for different file managers) by pressing Ctrl+H.

Up to the Point

I’ve had my dotfiles for quite some time. Every once in a while, I used to refactor them, change how they look, or completely change the way they were linked. More recently, I noticed that my shell was pretty slow on startup. I used Oh-My-Zsh! back then which was awesome but, as mentioned, a bit slow. I then wanted to give Prezto a try. Prezto was more convenient for me and faster but still it contained a lot of stuff that I never used. Eventually, I ended up writing my Zsh configuration from scratch.

There’s a kind of philosophy here between whether you should fork someone else’s dotfiles and build up from there or start from scratch. However, it’s all about personal taste and needs.

Good Tools Do Not Make a Good Carpenter

After I had made a lot of changes and rewritten almost everything, it kind of felt like I became obsessed with my dotfiles and how they looked instead of actually getting the real stuff done (A.K.A., using them for programming). This is a slippery slope. However important it is to prepare and sharpen the tools needed for a job, it should not replace the actual work.

الشافعي

There’s a story told about Al-Shafi‘i, a Muslim scholar, jurist, and poet who lived in the 8th century, that he had to take long walks in order to get to his lectures and classes. He said “How can I focus on my studies if I don’t have a donkey to ride?!” and bought himself a donkey. The story goes that after sometime, Al-Shafi‘i became very occupied taking care of his donkey cleaning and feeding it so he said: “How can I focus on my studies if I have a donkey?!”.
